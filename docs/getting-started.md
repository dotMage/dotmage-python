# Getting started

This page walks from installation to a first working vault, then covers unlocking an existing
device and the choice between the synchronous and asynchronous clients.

See also: [index](index.md) · [security model](security-model.md) · [API reference](api-reference.md).

## Install

```bash
pip install dotmage
```

The SDK requires Python `>=3.10`. Its only runtime dependencies are `httpx`, `tenacity`,
`pydantic` / `pydantic-settings`, `loguru`, and `PyNaCl` (libsodium bindings, used for all
cryptography).

## Configuration via environment

Configuration is read from environment variables with the prefix `DOTMAGE_` (a local `.env`
file is also honoured). See [`settings`](modules/settings.md) for details.

| Variable | Type | Description |
|----------|------|-------------|
| `DOTMAGE_SERVER_URL` | str | Base URL of the dotMage server |
| `DOTMAGE_DEVICE_TOKEN` | secret | Device token (`dmage_dtok_...`) — seeds an in-memory store |
| `DOTMAGE_REFRESH_TOKEN` | secret | Refresh token (`dmage_rtok_...`) |
| `DOTMAGE_MASTER_PASSWORD` | secret | Master password (never sent to the server; used to unlock) |
| `DOTMAGE_TIMEOUT` | float | Per-request timeout in seconds (default `10.0`) |
| `DOTMAGE_MAX_RETRIES` | int | Max transport attempts (default `5`) |
| `DOTMAGE_LOG_LEVEL` | str | loguru log level (default `INFO`) |

When you construct a client without an explicit `store`, the `DEVICE_TOKEN` / `REFRESH_TOKEN`
settings seed an in-memory [`MemoryStore`](modules/credentials.md). To persist tokens to disk
between runs, pass a [`FileStore`](modules/credentials.md) instead.

```python
from dotmage import DotMage
from dotmage import FileStore

dm = DotMage(store=FileStore())   # reads DOTMAGE_SERVER_URL from the environment
```

## First scenario: create a vault and push secrets

`init_vault` bootstraps a brand-new server (first device). It generates a random account key,
wraps it under your master password, registers the first device, and returns the client
already unlocked plus a one-time **recovery code**.

```python
from dotmage import DotMage

dm, recovery_code = DotMage.init_vault(
    "https://secrets.example.com",
    bootstrap_secret="XXXXXXXXXXXX",           # printed by the server on first boot
    master_password="correct horse battery staple",
    with_recovery=True,                         # default; generates recovery_code
    device_name="laptop",
)

# The recovery code is shown exactly once — store it somewhere safe and offline.
print("Recovery code (save this):", recovery_code)

dm.create_app("work/api")                       # app names may contain slashes (folders)
dm.create_env("work/api", "prod")

dm.push("work/api", "prod", {
    "DATABASE_URL": "postgres://...",
    "STRIPE_KEY": "sk_live_...",
})

secrets = dm.pull("work/api", "prod")           # -> dict[str, str], decrypted locally
print(secrets["DATABASE_URL"])

dm.close()                                       # or use `with DotMage(...) as dm:`
```

`push` and `pull` are the core of the SDK. `push` serialises the dict to canonical JSON,
encrypts it under the account key, and uploads only the ciphertext blob plus a content hash.
`pull` fetches that blob and decrypts it locally. The server never sees plaintext — see the
[security model](security-model.md) and the [crypto contract](crypto.md).

`push` accepts either a `dict[str, str]` or raw `.env` text (parsed via
[`dotenv`](modules/dotenv.md)). Convenience wrappers exist for files:

```python
dm.push_from_file("work/api", "prod", ".env")
dm.pull_to_file("work/api", "prod", ".env.prod")
env_text = dm.pull_text("work/api", "prod")     # serialised .env string
dm.set("work/api", "prod", {"NEW_KEY": "value"}) # merge into latest, then push
```

## Unlock an existing device

Once a device is enrolled, its tokens live in your credential store; you only need to
**unlock** to recover the account key into memory. Unlocking fetches the wrapped key from
`GET /api/v1/account/keys` and unwraps it locally with your master password:

```python
from dotmage import DotMage, FileStore

dm = DotMage("https://secrets.example.com", store=FileStore())
dm.unlock("correct horse battery staple")
print(dm.is_unlocked)                            # True
secrets = dm.pull("work/api", "prod")
```

If you have lost the master password but kept the recovery code:

```python
dm.unlock_with_recovery("ABCDE-FGHIJ-KLMNO-PQRST-UVWXY-Z2345")
```

Other ways to get an unlocked client:

- **`DotMage.enroll(server_url, enroll_token, master_password)`** — register this machine as a
  new device from an enrollment token (generated by `gen_enroll_token`), then unlock.
- **`DotMage.from_ci(server_url, ci_token, master_password)`** — build a client from a scoped
  CI device token (generated by `gen_ci_token`) and unlock. Ideal for pipelines:

  ```python
  import os
  from dotmage import DotMage

  dm = DotMage.from_ci(
      os.environ["DOTMAGE_SERVER_URL"],
      os.environ["DOTMAGE_DEVICE_TOKEN"],
      os.environ["DOTMAGE_MASTER_PASSWORD"],
  )
  dm.pull_to_file("work/api", "prod", ".env")
  ```

- **`DotMage.join(server_url, invitation_id, redeem_secret, master_password)`** — join a team
  from an invitation, re-wrapping the sealed account key under your own master password.

`lock()` forgets the in-memory account key without dropping the tokens; `close()` locks and
releases the HTTP connection.

## Sync vs async

`DotMage` (synchronous) is the primary entry point. `AsyncDotMage` is a mirror with the same
method names and signatures — every method that performs network I/O is a coroutine, and the
classmethods (`init_vault`, `enroll`, `join`, `from_ci`) are awaited. All cryptography,
diff/drift logic, endpoint specs, and session state are shared between the two.

```python
import asyncio
from dotmage import AsyncDotMage

async def main() -> None:
    async with AsyncDotMage("https://secrets.example.com") as dm:
        await dm.unlock("correct horse battery staple")
        secrets = await dm.pull("work/api", "prod")
        print(secrets)

asyncio.run(main())
```

Notes on the async client:

- Use `async with ...` (calls `aclose()`) or `await dm.aclose()` — not `close()`.
- `lock()` is synchronous on both clients (it only clears in-memory state).
- The `progress` callback passed to `rotate(...)` is a plain synchronous callable on both.

Continue to the [API reference](api-reference.md) for every method, or the per-module pages
starting with [`client`](modules/client.md) / [`async_client`](modules/async_client.md).
